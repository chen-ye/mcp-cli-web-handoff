# Product Guidelines: mcp-gemini-cli-web-handoff

## Prose Style
- **Tone:** Professional, clear, and direct. Avoid overly conversational or technical jargon where possible.
- **Clarity:** Use simple, concise language to explain complex technical concepts, such as WebSocket authentication or suspension loops.
- **Consistency:** Ensure consistent terminology is used across the CLI, MCP server, and Chrome Extension.
- **Accessibility:** Write content that is easy to understand for users with varying levels of technical expertise.

## Branding
- **Name:** `mcp-gemini-cli-web-handoff`
- **Voice:** The product's voice should be authoritative yet helpful, guiding the user through the handoff process.
- **Visual Identity:** The Chrome Extension's Side Panel should follow a clean, modern aesthetic that aligns with the Google Gemini web interface's visual style.
- **Logos & Icons:** Use clear, recognizable icons for buttons and status indicators to enhance user understanding.

## UX Principles
- **User-Centric Design:** Prioritize the user's needs and workflows. Ensure the handoff process is as seamless and intuitive as possible.
- **Feedback:** Provide clear, immediate feedback for all user actions (e.g., successful copy, connection status, response received).
- **Error Handling:** Gracefully handle errors and provide informative, actionable error messages to the user.
- **Efficiency:** Minimize the number of steps required for the user to transition between the CLI and the web interface.
- **Security by Design:** Integrate security measures (e.g., token-based authentication) into the core UX to protect user data and maintain trust.
